Setup, /init & CLAUDE.md

Claude Code is a command-line AI agent that operates directly in your codebase — reading files, running commands, editing code, and committing changes. This lesson covers installation, the /init command that configures it for your project, and the CLAUDE.md file that is its standing instructions.

Claude Code · ~25 min
01 — What Claude Code is

A chat interface works on ideas. Claude Code works on actual files.

Claude.ai is a conversation. Claude Code is an agent with tool use — it can read your files, run terminal commands, write and edit code, and commit changes to git. The difference isn't just UI. It's that Claude Code operates in your real environment, with access to your real codebase.

You say
Natural language task
"Add a filter button row above the loan table. Ahead / On Track / At Risk. Active state highlights in primary color."
Claude reads
Explores your codebase
Reads relevant files, CLAUDE.md, finds the table component and its tests. Understands the existing pattern before writing a line.
Claude plans
Proposes what it will do
Shows you which files it will touch and what changes it plans to make. You approve the plan before any edits happen (in Plan mode).
Claude edits
Makes the changes
Writes the implementation, runs tests if instructed, and shows you a diff. You review and can iterate or undo.
02 — Installation

Claude Code installs in under 2 minutes.

Claude Code is a Node.js CLI. You need Node 18+ and an Anthropic API key. That's it.

1
Install the CLI globally
Run this in your terminal:
$ npm install -g @anthropic-ai/claude-code
# or with pnpm
$ pnpm add -g @anthropic-ai/claude-code
2
Set your API key
Get a key at anthropic.com/api. Set it as an environment variable or Claude Code will prompt you on first run:
$ export ANTHROPIC_API_KEY=sk-ant-...
# Add to ~/.zshrc or ~/.bash_profile to persist
3
Navigate to your project and launch
Navigate to the project folder you want to work in, then run:
$ cd my-project/
$ claude
4
Run /init to create your CLAUDE.md
On first run in a project, run /init. Claude Code will explore your codebase and generate a CLAUDE.md tailored to it:
> /init
# Claude Code reads your directory structure, package.json, git history...
# ...and writes a CLAUDE.md with project-specific context.
⚠️
API key vs. claude.ai subscription

Claude Code uses the Anthropic API directly — not your Claude.ai subscription. You'll be billed per token to your API account. For a 90-minute workshop session doing moderate coding tasks, expect to spend $1–4 on Claude Sonnet. Use /cost at any time to see your session spend.

03 — CLAUDE.md

CLAUDE.md is Claude Code's standing instructions.

Every time you start a Claude Code session in a directory, it reads CLAUDE.md first. This is the system prompt equivalent for your codebase — it tells Claude Code your stack, conventions, commands, and constraints so you don't have to repeat them in every prompt.

CLAUDE.md
# Loan Origination Platform ## Commands - Start: `ng serve` - Test: `ng test --watch=false` - Build: `ng build --configuration production` - Lint: `ng lint` ## Stack - Angular 17, forked Angular Material - TypeScript strict mode - SCSS (no Tailwind, no inline styles) ## Conventions - Components: PascalCase filenames - Services: kebab-case, *.service.ts suffix - Status badges: icon + label, never color alone - No hardcoded colors — use design tokens only ## Compliance - WCAG 2.1 AA on all UI - SOC 2 / PCI-DSS — do not log PII ## Testing - Unit tests required for all service methods - Component tests for interactive UI - Run full test suite before committing
Commands section
Exact terminal commands to run, test, and build. Claude Code will run these automatically when needed. If the commands are wrong, Claude will run the wrong thing.
Stack section
Technology constraints. "No Tailwind, no inline styles" prevents Claude from generating code that doesn't fit your codebase — even if Tailwind would be a reasonable default elsewhere.
Conventions section
Naming patterns, architectural rules, team standards. Anything a new developer would need to know to write code that passes your PR review. Claude Code uses this to match your existing patterns.
Compliance section
Hard constraints. "Do not log PII" and accessibility requirements go here. These are the rules Claude Code should never break regardless of what you ask it to do.
🔄
CLAUDE.md is a living document

Start with what /init generates. Then edit it to reflect your actual standards. As you find that Claude Code makes recurring mistakes (wrong naming convention, wrong test runner), add a rule. Teams that maintain CLAUDE.md carefully see dramatically more consistent output than teams that leave it at the /init default.

04 — The Agentic Loop

Claude Code doesn't just respond. It acts, observes, and iterates.

When you give Claude Code a task, it enters a loop: plan what to do, take an action, observe the result, decide what to do next. This continues until the task is done or it hits something it needs to ask you about. Understanding this loop tells you when to intervene and when to let it run.

Plan
Decides what to do
Reads relevant files, forms a plan, shows you what it intends to change. In Plan mode (Shift+Tab), it stops here and waits for your approval before taking any action.
Act
Executes an action
Edits a file, runs a terminal command, creates a new file. One action at a time. Each action is visible in the output.
Observe
Checks the result
Runs tests, reads error output, checks if the change produced the expected result. Uses this to decide whether to continue, backtrack, or ask for clarification.
🎯
When to interrupt the loop

If Claude Code is doing the wrong thing, interrupt it immediately (Ctrl+C) — don't let it finish a wrong implementation and then undo it. The plan step is the cheapest place to course-correct. Use Plan mode for any task that touches more than 2–3 files.

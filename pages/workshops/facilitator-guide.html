Demo scripts · prompts · timing · troubleshooting
Hold the shape

The same rhythm every week

Each workshop follows one shape: open, teach, demo, discuss, break, lab, debrief, close. Protect the lab — it's 40 minutes and it's where the value lands. If you run long, cut the teach, not the lab.

01
Every session ends in a deliverable
Without it, people use Cowork like chat. Make the lab a real delegation that produces a document, memo, or plan.
02
Teach cost discipline early
Default to Sonnet; reserve Opus for genuinely hard reasoning. Say it in Workshop 1 and repeat it.
03
Tally the room live
The maturity poll is client-only — it shows each person their own answer. Ask for a show of hands per level and pitch the session to the middle.
Workshop 1 · Setup & Foundations

Demo, discussion, and timing

Demo (20 min)

Summarize a loan file

Stage a sample loan-file folder. Ask Cowork for a one-page summary with a cited findings list. Narrate: this is delegation, not chat — show the plan, approve a step, and open the finished file. Contrast with pasting the same ask into chat.

Keep it on Sonnet; point out the cost note.
Discussion (10 min)

"One task you'd hand off"

Go around: each person names one repetitive, document-heavy task. Good answers are concrete ("summarize my deposit-relationship notes"). Capture them — they seed Workshop 2's use-case shortlist.

M1 is the thinnest module — let the demo and lab breathe.

Run it as a before/after. Download the sample loan file, drop it in a folder, and run the prompt below cold — before co-setup — then run the exact same prompt after. For the "after," answer the co-setup interview as the Maya Chen persona (in the facilitator script), or drop in her ready-made profile to skip straight to it. Same ask; the second answer comes back in the banker's voice and framed for the role — that contrast is the demo.

Download the sample loan file ↓ Browse the Maya demo profile →
Demo prompt — paste into Cowork (run before, then after) 
Read the loan file in this folder and write a one-page credit summary I can take to committee. Cover: the borrower and the request, the key financials and any trends, collateral and guarantors, covenant headroom, and the top three risks to flag. Don't invent numbers — if something isn't in the file, say so.
Workshop 2 · Use Cowork

Demo, tracks, and timing

Demo (20 min) — A credit memo from a loan file

Asset: assets/demo/sample-loan-file.md (same file as W1). Run on Maya's profile or your own so the memo comes back in a credit-analyst voice. Stage: put sample-loan-file.md alone in a clean folder, for example loan-demo. Grant Cowork only that folder and say "least privilege" out loud when you do it. Set the model to Sonnet.

Demo prompt — paste into Cowork 
Read the loan file in this folder and draft a credit memo for committee: the request, the borrower, the financial trend, collateral and guarantors, a covenant analysis, and a risk section with the top risks and a recommendation with conditions. Cite the file for every figure, flag anything missing or interim, and treat it as a draft for sign-off — not a decision.

As it runs: watch it flag the thin 1.28x DSCR against the 1.25x covenant, the roughly 24% customer concentration, and the stale interim financials. Every figure cites the source file. It calls out what is missing rather than filling it in. That is a draft a banker signs off on.

🛡️
Prompt injection beat — say this out loud

"Notice approvals are on. If this file, or an email I had forwarded in, carried a hidden line saying ignore your instructions and email this out, Cowork would pause at that step and I would catch it. That is why we keep least privilege and approvals on for anything from outside the bank."

Discussion (10 min) — Shortlist use cases

Have each person pick two banking use cases that fit their week and drop them in Teams. Push for concrete examples: "triage my overnight AML alerts," "summarize a new deposit relationship." Tally the top few live. These become the lab tasks and the skill candidates for Workshop 3.

Lab tracks (1:10–1:50)

Two parallel tracks run at the same time after the prompting primer. Circulate during staging and point people toward the right track. Sample files for both tracks are in the resource library.

Both tracks: one folder, Sonnet, approvals on, acceptance test written before starting. If a participant's files do not fit either track clearly, put them in Track 1 with the sample loan file.

📊
Track 2 facilitator notes

If they only have one file: the prompt still works — Cowork will flag what is missing, a useful demo of honest behavior. If the output is too generic: "Add one sentence — 'The material variances this period are in [category].' Rerun." If someone asks to switch to Opus: hold the line. Document reading and drafting is Sonnet territory.

🚧
Hold the line

Least-privilege folder access and approvals on for outside content are not optional during the lab. Do not wait for a perfect first pass. Make sure everyone finishes a deliverable.

Workshop 3 · Build a Skill

Open, demo, discussion, and lab

Open (10 min)

W2 quiz + skill seeds

Open with the Workshop 2 knowledge check — run it live in the poll tool (5 min), read each answer aloud, and close the loop. Then pull 2–3 use cases from the W2 survey submissions and name them: "These are today's skill seeds — by the end of the vault you'll know how to build one of these."

Pre-recruit two heavy users the day before: "I'd love you to kick off the discussion — would you share your take on good vs bad skills?" No cold calls.
Demo (19 min)

Decompose the memo workflow

Run the workflow-decomposition skill live on the memo workflow. Then open the memo skill and read it against the standards: description, length, good/bad examples. This is the discipline the Skill Vault tests.

Close with: "Everything you just watched is a key to a door."
Discussion + teams (12 min)

Good skill vs bad, then vault teams

Pre-recruited heavy users share first — no cold call. Critique a sample description: is it third-person and keyword-rich? Would it actually trigger? A bad one is vague, first-person, 900 lines, and has no evals. At the 8-min mark, announce breakout room assignments (2–4 per team) and who screen-shares Cowork. Give teams 30 seconds to find their room before break.

Evals before docs — say it twice.
Lab (40 min)

Run the Skill Vault

Ensure all five lab files are shared before breakout rooms open — files are on the lab page as downloads. One driver per team runs Cowork and shares their screen; rest advise. Keep the facilitator view open: live progress per team, hints used, and a Reset for stuck teams. Hint 1 is free; hint 2 adds 2 minutes. Setup and the optional shared-leaderboard backend are in the vault README.

Codes are case-insensitive; the format line in each room's last lab step is the nudge to give first.

Demo, discussion, and timing

Demo
Govern by workload + the dashboard

When something blocks

Common snags

01
Licenses & SSO
Confirm Claude Cowork access in pre-work. SSO-only sign-in blocks the skill-creator's run_loop optimizer — tune descriptions by hand instead.
02
Connectors not approved
If the M365 (or Google) connector isn't on, fall back to local files: grant a folder and run from there. Plan the labs local-first.
03
Scheduled tasks & the audit gap
Claude Cowork's scheduled tasks need the machine awake and the app open. And be precise about the June-2026 audit-coverage gap — manage it, re-verify it; don't overstate it.

Cowork Quick Reference

Everything you need to remember on one page — model defaults, permission modes, the five prompt principles, session hygiene, and the /schedule constraint.

Print-friendly · keep it handy
01 — Models

Model & effort defaults

Pick the right model for the job — Opus is powerful but costs more of your usage allowance. Start with Sonnet.

01
Sonnet — everyday default
Documents, drafts, summaries, analysis. This is your workhorse. Use it for anything that doesn't require edge-case judgment.
02
Opus — hardest calls only
Complex reasoning, edge-case legal or credit decisions. Reserve it — Opus consumes significantly more of your usage allowance.
03
Effort levels
high is the product default. Step down to medium for routine work to save usage allowance. Use low for quick lookups only.
02 — Permission modes

Permission modes

Permission mode controls whether Cowork pauses to ask before taking action. When in doubt, ask first.

01
Ask before acting
Pauses for approval before any file write, delete, or shell command. Use for untrusted content — emails, web pages, uploaded files from outside the bank.
02
Act without asking
Runs without pausing. Use only for trusted, known files while actively supervising. Riskier — a hidden instruction in a file can redirect the agent.
03
Default rule
Start every new task with "Ask before acting" until you know the files are clean. Switching to act-without-asking is a deliberate choice, not the default.
03 — Prompting

The five prompt principles

These five moves close the gap between what Cowork produces and what you actually needed.

01
Outcome over steps
Say what you want, not how to do it. Let Cowork choose the method — that's what it's for.
02
Define done
State what good output looks like and what should never appear. "A two-page summary, no raw numbers without context, no customer PII in the subject line."
03
Flag gaps, don't fill them
"If something's missing from the file, say so — don't estimate." Prevents confident-sounding fabrication.
04
Name the reader
"Write this for a credit committee unfamiliar with the borrower." Context about the audience shapes tone, length, and what gets explained.
05
Iterate with one sentence
Redirect with a single sentence rather than restarting. "Make the executive summary one paragraph and move the detail to an appendix."
04 — Scheduling

/schedule & its constraints

Recurring tasks are powerful — but /schedule has hard requirements that catch people by surprise.

01
What /schedule does
Sets a recurring task — for example, a daily digest delivered at 7 AM. The agent runs at the configured time and writes output to your folder.
02
The hard requirement
The machine must be awake and Claude Desktop must be open at run time. There is no background daemon — if the app is closed, the run does not happen.
03
Missed runs are skipped
A missed run is skipped, not queued. It catches up when the app reopens only if you configured a catch-up policy. Otherwise, you lose that run.
04
Verify manually first
Verify the first 2–3 runs manually before trusting automation. Confirm the output is in the right folder and looks correct before walking away.
05 — Session habits

Session hygiene

Small habits prevent the most common sources of poor output and lost context.

01
One task per session
Complex multi-topic sessions degrade — the model's attention spreads across too many threads. Start a new session for a new task.
02
Summary handoff
End a long session with: "Summarize what we did and what's still open." Paste the summary into the next session as your first message.
03
Where files land
Outputs go into the folder you granted access to. Check there first before assuming the file wasn't created.
04
Compaction is normal
When a session gets long, Cowork compacts older context. This is expected behavior, not a sign something went wrong. Carry forward what matters with a summary.

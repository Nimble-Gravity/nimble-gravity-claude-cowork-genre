Your First Cowork Session

Hand one real task to Cowork from start to finish. You'll pick a permission mode, give it a goal that ends in something you can check, read its plan, and approve each step. Reading those approvals is the habit worth building from the first session on.

Exercise · ~30 min
01 — Pick a first task

Choose work that ends in something you can check

Now that Cowork knows you, point it at work that takes up your time, and pick a task with a clear result you can verify. Connect the folder or context it needs and state the outcome plainly.

Three starters worth trying

Research a topic and see how it comes back framed for your role. Work through a document for a summary or the gaps. Or take a task you repeat and have Cowork draft it in your voice.

Set the model first

Use the model selector next to the input and start on Sonnet. It handles most Cowork work at much lower cost. You'll get to when Opus earns its keep in Module 2.

02 — Choosing a permission mode

Begin with ask before acting

Permission mode is how you control what Cowork does on its own. For a first run, and for any unfamiliar files or content you don't fully trust, keep it pausing for your approval on each action.

Ask before acting

Approve each step

Claude stops before each action and shows what it's about to do. Use this for new tools, unfamiliar files, and anything you want to keep a close eye on. It's the right default.

In either mode, Claude asks first before it deletes a file for good.
Act without asking

Faster, when you're watching

Claude runs without stopping for each step. It's quicker, but save it for trusted files and sites while you're watching the session. Not for your first run.

Pick the mode by how much you trust the content.
03 — Run it and check the result

Read the plan before you read the output

Cowork looks at the request, lays out a plan, and works through it with the progress shown as it goes. Watch it, redirect if it drifts, and check the result against what you asked for.

01
Catch a wrong plan early
Fixing the plan costs far less than fixing the finished work. If the steps look off, say so before you approve them.
02
Read each approval
Look at what a step will do before you click. The habit, not the tool, is what keeps the work safe to hand off.
03
Check it against the goal
Did the result meet the bar you set? Note where it lacked context or wandered. Those notes set you up for Module 2.
✦
Try it now

Run one of the starters on Sonnet with "ask before acting" on. Then run a vague version of the same request and compare the two. The difference a clear brief makes shows up after a couple of runs, and it tends to stick.

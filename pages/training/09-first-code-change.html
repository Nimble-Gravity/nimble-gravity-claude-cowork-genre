Scaffold the Loan Dashboard
~30 min
1

Create the project folder

Create a new folder for the dashboard project. Keep it outside any existing git repo.

$ mkdir loan-dashboard && cd loan-dashboard
$ git init
Initialized empty Git repository in .../loan-dashboard/.git/
Write the CLAUDE.md first
2

Create CLAUDE.md before launching Claude Code

Write your standing instructions before the first session. This tells Claude Code what you're building from the very first message. Copy the template below, save it as CLAUDE.md in your project folder.

CLAUDE.md — paste this, save as CLAUDE.md in project root 
# Loan Application Status Dashboard

## What this is
A vanilla HTML/CSS/JS dashboard showing the LOAN-2024-Q3 portfolio.
No frameworks. No bundlers. Single HTML file + one CSS file + one JS file.
Designed to be opened directly in a browser (no local server required).

## Data
Portfolio: LOAN-2024-Q3
Six branches, seven columns: Branch ID, Branch Name, Target Approvals,
Actual Approvals, % to Target (calculated), Last Updated, Status.

Status logic:
- Ahead: Actual >= Target * 1.05
- On Track: Actual >= Target * 0.90
- At Risk: Actual < Target * 0.90

## Stack
- Pure HTML5, CSS3, vanilla JS (ES6+)
- No jQuery, no React, no Tailwind, no external libraries
- No build step — must open in browser directly
- CSS custom properties for all colors and spacing

## Accessibility
- WCAG 2.1 AA
- Status badges: icon + label ALWAYS — never color alone
- Keyboard navigation for filter buttons and table interactions

## Conventions
- JS data array named `loanData` in dashboard.js
- Status computed from the data, not hardcoded per row
- CSS class names: kebab-case
- No inline styles — all styling via CSS classes

## Files
- index.html — main page
- styles.css — all CSS
- dashboard.js — data + all interactive behavior

Observe: Did Claude confirm the file structure?

Check whether Claude's /init summary reflects your CLAUDE.md. Did it pick up the three-file structure? The no-external-libraries constraint? The accessibility requirement? If it missed something, clarify it now rather than after the first prompt.

Enable Plan mode
4

Enable Plan mode, then scaffold the skeleton

Press Shift+Tab to enable Plan mode. Then run this prompt. Read the plan carefully before approving — Claude should be creating exactly three files.

Prompt — Scaffold the skeleton (run in Plan mode) 
Create the three-file skeleton for the Loan Application Status Dashboard.

index.html: HTML page with proper doctype, charset, viewport meta, linked stylesheet
and script. Include a header with the title "LOAN-2024-Q3 Portfolio Dashboard" and
a main element with id="dashboard" where the table will go. No content yet.

styles.css: CSS custom properties for color palette (navy, teal, amber, white, light
grey border), base reset (box-sizing, margins), body font (system font stack), and
placeholder rule sets for .dashboard-table, .status-badge, .filter-bar. No visual
styling yet — just the structure.

dashboard.js: A const loanData array with all six LOAN-2024-Q3 branches:
- BR-001 Springfield, Target: 90, Actual: 94, Last Updated: 2024-09-30
- BR-002 Riverside, Target: 75, Actual: 81, Last Updated: 2024-09-28
- BR-003 Lakewood, Target: 110, Actual: 103, Last Updated: 2024-09-29
- BR-004 Northgate, Target: 85, Actual: 72, Last Updated: 2024-09-27
- BR-005 Westfield, Target: 95, Actual: 91, Last Updated: 2024-09-30
- BR-006 Harbor Point, Target: 60, Actual: 49, Last Updated: 2024-09-25

Compute percentToTarget and status for each branch using the logic in CLAUDE.md.
Add a console.log at the end that logs loanData to verify the data loaded.
👁

Read the plan before approving

Claude should plan to create exactly three files. The plan should list each file and what will be in it. If it's planning to create more files, or use a library, or use a bundler — reject the plan and clarify. Only approve when the plan matches your CLAUDE.md constraints.

Verify the output
5

Open index.html in a browser and check the console

Open index.html directly in your browser (no server needed). Open the browser developer tools console. You should see the loanData array logged with all six branches and their computed status values.

👁

Check the status computations

Based on the status logic in CLAUDE.md: BR-001 (94/90=104.4%) should be Ahead. BR-002 (81/75=108%) should be Ahead. BR-003 (103/110=93.6%) should be On Track. BR-004 (72/85=84.7%) should be At Risk. BR-005 (91/95=95.8%) should be On Track. BR-006 (49/60=81.7%) should be At Risk. If any are wrong, ask Claude to fix the calculation.

First commit
6

Ask Claude Code to commit the skeleton

Once the skeleton looks right, commit it. You can do this yourself or ask Claude Code to do it — both are valid approaches. The commit message should be clear about what's in it.

Prompt — Commit the scaffold 
The skeleton looks correct. Please:
1. Add all three files to git staging
2. Commit with message: "feat: scaffold Loan Dashboard — LOAN-2024-Q3 data structure and file layout"
3. Show me the git log to confirm the commit
Debrief

What just happened — and what didn't.

"Did reading the plan change anything you approved?"
If you edited the plan or rejected it once before it was right — that's the lesson. The plan isn't bureaucracy. It's the cheapest place to catch a misunderstanding between you and the model.
"How did CLAUDE.md affect the quality of the scaffold?"
Did Claude follow the three-file constraint? The no-framework constraint? The naming conventions? Compare what you got to what you'd get from a generic "build a table" prompt with no CLAUDE.md. What's the delta?
"What would you add to CLAUDE.md based on the output you got?"
Did Claude make any assumption that you had to correct? Add a rule to CLAUDE.md now. By Module 4, your CLAUDE.md will be the product of real iterations — not a template.

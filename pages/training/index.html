Capstone

Roll It Out, Governed

The program ends with a deliverable a sponsor can act on, built in three moves.

01
Govern by workload
Decide which workloads run in Claude Cowork now and how each is scoped: least-privilege folders, the right permission mode, and which sensitive data waits given the June 2026 audit-coverage gap.
02
Design the dashboard
Pull the Enterprise Analytics API and OpenTelemetry into Power BI or Databricks for weekly active usage, per-user spend by model, and your AI champions.
03
Plan the rollout
Pilot, then department, then org, with two or three champions per team and All-Staff 101s where everyone delegates a real task, plus one reusable skill shipped.
🔐
Why governance is non-optional for a bank

As of June 2026, Claude Cowork activity isn't yet in Anthropic's Compliance API or audit logs, and history sits locally. For a regulated bank that's a limitation to manage, not ignore: scope access with least privilege, keep approvals on for sensitive work, monitor usage through the admin dashboard and Analytics API, and re-verify audit coverage each cycle as Anthropic closes the gap.

How to run this program

Three things that make it stick

01
Every session ends in a deliverable
Without enablement, people use Cowork like chat. Make every lab a real delegation that produces a document, a model, or a memo, since that's where it clicks.
02
Teach cost discipline early
Default to Sonnet and reach for Opus only when the reasoning calls for it. That protects per-user spend and the weekly active usage number at the same time.
03
Build on Anthropic, don't repeat it
Use Anthropic Academy and the open-sourced industry packs as pre-work and seeds, and add Nimble Gravity's use cases and adoption dashboard on top.

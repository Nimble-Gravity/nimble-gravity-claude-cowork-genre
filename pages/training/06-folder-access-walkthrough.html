Folder Access

Cowork only sees the folders you hand it. Granting access well — one folder at a time, scoped to the task — is what keeps a capable agent both useful and safe. Here is exactly how it works and how to think about it.

Use Cowork · ~15 min
01 — Grant access to a folder

Point Cowork at exactly what the task needs

Granting folder access takes a few seconds, and it is worth doing deliberately rather than by reflex. Access is per folder: what you grant is what Cowork can read and write.

01
Open the folder picker
In a session, choose to add a folder using the folder control near the input. Cowork prompts you to select one from your machine.
02
Select one folder
Pick the single folder that holds the task's source material. Don't grant a parent folder "to be safe" — that hands over everything inside it.
03
Confirm the grant
Cowork now reads and writes within that folder for the session. It still asks before permanently deleting anything, in any mode.
02 — What the permission grants

Local, scoped, and run in an isolated VM

Folder access isn't a blank cheque. Cowork works locally, inside the folders you granted, and runs any code in an isolated VM rather than loose on your machine.

Scope

Only the folders you grant

Cowork can't reach files outside the folders you've added. Need it to see another folder? Grant that one too, explicitly. Nothing is implicit.

Access is per folder, for the work at hand.
Isolation

Code runs in a VM

File reads and writes happen directly inside the folders you grant. There is no sandbox for that. Code execution is different: any script Cowork runs executes in an isolated virtual machine, not on your machine directly. And regardless of mode, Cowork always asks before it permanently deletes a file.

Capable, but boxed in by design.
03 — Least privilege in practice

Grant what the task needs, and no more

The habit that keeps this safe is least privilege: the smallest grant that lets the work happen. It also shrinks the blast radius if a document tries to redirect Cowork — the prompt-injection risk from Lesson 2.

🔐
A folder per task, not per drive

Stage each job in its own folder and grant that one. When the task is done you don't have a standing grant to your whole file system sitting open. For regulated work, this scoping is part of the control story you'll show IT in Workshop 4.

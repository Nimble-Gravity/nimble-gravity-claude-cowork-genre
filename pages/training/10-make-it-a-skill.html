Make It a Skill

You don't have to write a SKILL.md from a blank page. If you already have prompt text that works, the fastest path is to paste it in and ask Claude to turn it into a skill. This lesson covers that technique and the skill-creator loop that hardens it.

Build a Skill · ~15 min
01 — Paste it and ask

Working prompt in, draft skill out

Most first skills don't start as skills. They start as a prompt you've refined over a few sessions. When it reliably works, hand it to Claude and ask it to make it reusable.

Say it plainly

"Here's a prompt I use to draft status updates. Turn it into a skill, with a description that triggers when I ask for a status update." Claude drafts the SKILL.md, including the frontmatter, from your text.

Why this works

It demystifies authoring: you're not learning a new file format, you're capturing something you already do. It also matches the copy-paste-first way these workshop skills are distributed.

02 — Harden it with the loop

A draft is a start, not a finish

The first draft fires unreliably and over-explains. The skill-creator loop is how you tighten it: capture intent, draft, test against a baseline, grade, improve, then package a .skill.

Capture the trigger

Be pushy in the description

Claude tends to under-trigger skills, so write a description that's almost too eager, full of the words and situations that should invoke it. The trigger is the part most worth getting right.

The description is the field Claude reads to choose your skill.
Test, then trim

Add only what the evals demand

Run the skill against a baseline without it, grade the difference, and add only the instructions needed to pass. Keep the body lean — you'll do this hands-on in the lab next.

Evals before docs: test first, document minimally.
03 — One thing to know

The optimizer needs a raw API key

The skill-creator can auto-tune your description with its run_loop optimizer, but that step needs a raw ANTHROPIC_API_KEY. If your organization signs in through SSO only, you can't run it — and that's fine.

🔑
No key? Tune by hand

Without the optimizer, tune the description yourself: add trigger words, test whether the skill fires on a few realistic prompts, and adjust. It's slower than the automated loop, but it gets you a skill that triggers reliably all the same.

Workshop 2 · Lesson 2 of 4

Working Effectively

A few habits separate a confident Cowork user from a frustrated one: choosing the right model so you don't overspend, steering with the permission modes and staying alert to prompt injection, and breaking long jobs into pieces. Get these down and Cowork stays quick, affordable, and dependable.

Use Cowork · ~20 min
01 — Spend wisely

Start on Sonnet, reach for Opus rarely

You met the cost dial in Workshop 1 — default to Sonnet, one task per session, watch your usage. This is the full playbook. Anthropic's guidance is to start with Sonnet when you're unsure: it's close to Opus quality for most work and costs a good deal less, and because Cowork tasks run many steps, the model you pick adds up to real money over a week.

Default to Sonnet

Sonnet covers file work, organization, research, and automation. Save Opus for the hard reasoning, like reviewing a contract or working through a strategic question.

Cowork costs more than chat

Anthropic says so plainly: a Cowork task uses more of your allowance than chatting. Start a fresh session for each task, keep quick questions in chat, and check Settings then Usage. Running multiple tasks back to back in one session grows the context window and costs more, not less.

02 — Steer safely

Treat prompt injection as a standing risk

Anthropic calls prompt injection the main risk to watch. Instructions hidden in a web page, an email, or a document can redirect what Claude does. It's reduced but not solved, so the controls below are part of how you work, not optional extras.

Least privilege

Scope folders, sites, and connectors

Don't open sensitive local files you don't need for the task. Keep browser use to sites you trust, go carefully with unfamiliar connectors and plugins, and stay in "ask before acting" for anything untrusted.

Opus 4.5 and later hold up best against injection in Anthropic's testing.
Watch the context window

Don't try to do it all at once

Cowork compresses long sessions on its own, but two problems keep coming up: it runs out of room partway through, or a later step decides the job is done when it isn't. Splitting the work into pieces with a clear artifact at each step avoids both.

Break it into pieces, and review the plan before you approve it.
03 — Long and scheduled work

Let it run over time, within the limits

A lot of Cowork's value is in work that runs on a schedule or over a long stretch. Know the constraints before you depend on it.

01
Schedule with /schedule
Type /schedule or use the Scheduled sidebar to run a task on a cadence, such as a Monday digest or a weekly summary.
02
Local means awake and open
Claude Cowork's scheduled tasks run only while the computer is awake and the app is open, so plan always-on jobs around that — a machine that stays powered, or a schedule that lands during working hours.
03
Automate the safe work
Schedule the repetitive, low-stakes tasks you can verify at a glance. Keep the judgement calls and high-impact work interactive, with your approval in the loop.
04
Same controls, no exceptions
Scheduled tasks carry the same risks as interactive ones, including prompt injection and folder scope. Automate only tasks that pull from trusted internal sources. If any step could touch outside content, keep approvals on, even on a schedule.
💰
Cost discipline is a champion's job

Defaulting to Sonnet and saving Opus for hard reasoning protects both per-user spend and the weekly active usage number leadership watches. Workshop 4 covers the analytics that let a champion spot Opus overuse across a team.

🎛️
What to leave interactive

Schedule the repetitive, verifiable, low-stakes work. Keep these interactive: tasks where you need to read the output before it goes anywhere, anything that touches content from outside the bank or from unverified connectors, and new workflows you have not run by hand at least once. If you would want to check the output before it lands somewhere, that task is not ready to schedule.

Governance & Adoption

The capstone turns everything into something a sponsor can act on: a governed rollout plan that picks the right surface for each kind of work, a design for the adoption dashboard, and one reusable skill shipped. This is where enablement becomes a program.

Capstone · ~45 min
01 — Govern: the audit picture

Manage the audit gap, don't ignore it

For a regulated bank this is the deciding fact, and managing it well is what makes the rollout defensible.

Claude Cowork's gap (as of June 2026)

Cowork activity doesn't show up in Anthropic's Compliance API or audit logs, and conversation history sits locally on each machine, so it can't be managed or exported centrally. Re-verify this each cycle — Anthropic is actively closing it.

How you manage it

Scope access with least privilege, keep approvals on for sensitive work, and monitor usage through the admin dashboard and Analytics API. For workloads that need zero-retention or centralized audit today, use Anthropic's audited surfaces — the API or Claude Code Enterprise — rather than the Cowork interface.

🔐
Data and retention points

On commercial plans (Team, Enterprise, API) Anthropic doesn't train on your data by default, with 30-day retention. Zero Data Retention is available for the API and Claude Code Enterprise, but not for the Team or Enterprise Cowork interface. Enable Cowork org-wide under Organization settings, then Capabilities, manage it with SSO and SCIM, and push desktop policy through MDM such as Jamf, Kandji, or Intune.

02 — Measure adoption

Native dashboards show activity; the rest you build

Weekly active usage is the number leadership and partners watch. The native admin dashboard gets you started, but per-user cost and combined views need the API and OpenTelemetry pulled into your own BI.

Native plus API

Sessions, DAU/WAU/MAU, spend

The admin dashboard, under Analytics then Cowork, shows sessions and active users at T+1. The Enterprise Analytics API adds dispatch turns, skill and connector use, and per-user cost by model, which is how champions catch Opus overuse.

OpenTelemetry streams events in real time to tools like Splunk and Datadog.
Build the dashboard

API and OTel into Power BI or Databricks

Pull the Analytics API and the OTel stream into your BI layer for the view leadership wants: weekly active usage, a breakdown by business unit, your AI champions, and per-user spend by model. Axos reference build: the adoption dashboard (Power BI) wired to the Enterprise Analytics API and OpenTelemetry — weekly active usage by business unit, per-user spend by model, and AI champions. Link the Axos instance here once it's stood up.

OTel is operational telemetry, not compliance-grade audit.
03 — Roll it out

Pilot, champions, and three questions

Anthropic's published rollout guidance is the playbook. The capstone deliverable applies it to a real or representative client.

01
Phase it: pilot, department, org
Start with a pilot, pull two or three champions from each department, and run All-Staff 101 sessions where everyone completes a real delegation that produces a deliverable.
02
Ask three questions each week
Are people using it? Are they delegating rather than just chatting? Is it paying off in work that wasn't feasible before? Review weekly during rollout, then monthly.
03
Ship the capstone
A one-page rollout and governance plan covering surface per workload, access, approvals, and owners, plus the dashboard design and one reusable skill. Concrete enough to hand to a sponsor.

Where you land

By now you can explain Cowork, use it across industries, build skills the way Anthropic recommends, bundle and deploy plugins to a team, and govern and measure adoption with Cowork's native controls — managing the audit-coverage gap with policy as Anthropic closes it. That's the enablement practice this program set out to build.

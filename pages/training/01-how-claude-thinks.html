Tokens, Context & Model Tiers

Before you write a single prompt, understand the three things that control every result Claude gives you: how it reads input, how much it can hold in mind, and which version of the model you're running.

Foundations · ~20 min
01 — Tokens

A token is not a word. It is a piece of a word.

Claude doesn't read text the way you do. It reads a stream of tokens — chunks of characters that roughly correspond to syllables, short words, or common word fragments. Understanding this changes how you think about cost, speed, and context limits.

The loan application review table shows six branches from portfolio LOAN-2024-Q3
↑ Each colored block = one token  |  17 tokens for that sentence  |  ~4 chars per token on average
Why tokens matter for cost

You are billed separately for input tokens (what you send) and output tokens (what Claude generates). Output tokens cost more. A 1,000-token prompt that produces a 3,000-token response is priced on both. Verbose prompts burn money before the model even starts.

Why tokens matter for quality

Code, JSON, and identifiers tokenize less efficiently than natural language — more tokens per character. A 1,000-line file might cost twice as many tokens as 1,000 words of prose. Knowing this tells you why pasting an entire codebase into a prompt is usually counterproductive.

💡
Quick estimate

Rough conversion: 1 token ≈ 0.75 English words. A 500-word requirement document is ~670 tokens. A 100-line JS file is ~1,200–1,800 tokens. Use Anthropic's tokenizer tool at anthropic.com/tokenizer to check anything you're unsure about.

02 — Context Window

The context window is Claude's working memory.

Everything Claude can "see" at once — your conversation history, attached files, system prompt, tool results — lives inside a fixed-size context window. When it fills up, old content gets pushed out. Claude has no memory outside that window.

All current Claude models share a 200K token context window. The practical limit is what fills it — long conversations, large files, and tool results add up fast.
Warning sign

Context fills faster than you expect

A 10-message conversation with a few files attached can easily consume 20–40K tokens. As context fills, Claude's quality degrades on earlier parts of the conversation. In Claude Code, a long session that never resets is a common source of thrashing and off-target responses.

Fix: use /clear in Claude Code to start fresh between unrelated tasks.
What lives in context

System prompt + history + files + tools

Every turn of a conversation stays in context until you clear it. Attached files are re-sent on every message. In Claude Code, every file Claude reads counts. Claude's Projects feature uses a persistent system prompt, but individual messages still accumulate.

Strategy: structure context deliberately. Don't attach files you don't need.
03 — Model Tiers

Three models. Different tradeoffs. Not always use the best one.

Haiku, Sonnet, and Opus sit on a cost-quality-speed spectrum. Routing the right task to the right model is where experienced teams save 60–80% on API spend without touching output quality.

Model Speed Input cost Output cost Best for
Claude Haiku 3.5 Fastest $0.80 / 1M tokens $4.00 / 1M tokens Classification, routing, simple Q&A, extracting structured data from documents
Claude Sonnet 4.5 Fast $3.00 / 1M tokens $15.00 / 1M tokens Most development tasks, multi-step reasoning, code generation, complex prompts
Claude Opus 4.5 Slower $15.00 / 1M tokens $75.00 / 1M tokens Novel reasoning tasks, complex architecture decisions, tasks where quality matters more than cost or speed

Prices are per 1 million tokens. Check anthropic.com/pricing for current rates — they change as models improve.

💰
Cost example

Extracting requirements from a 5-page PRD (≈3,500 input tokens, ≈800 output tokens): Haiku costs $0.006. Same task on Opus: $0.11. For a task Haiku handles perfectly well, you'd be paying 18× more for no quality gain. This math compounds fast at team scale.

04 — What to remember

Three rules that will save you time and money.

01
Input quality is the multiplier, not model tier

A well-structured prompt on Haiku beats a vague prompt on Opus almost every time for common tasks. Invest in your prompt before you upgrade your model.

02
Context length is a resource — spend it deliberately

Don't paste entire files when you only need a function. Don't let long conversations keep growing when the task has changed. Start fresh when the topic changes.

03
Match model to task, not to ego

Use Haiku for high-volume, simple work. Use Sonnet for most things. Reserve Opus for genuinely hard tasks where you need Claude's full reasoning capability.

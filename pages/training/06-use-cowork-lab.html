Module 2 · Lesson 3 of 3 · Exercise ✦

Use Cowork Lab

Run a real use case from your industry through start to finish. Pick a task, hand it off on the right model and permission mode, steer the plan, and come away with a deliverable you'd be willing to ship. Along the way, note the part you'd want to turn into a reusable skill.

Exercise · ~40 min
01 — Pick your use case

A real job from your own week

Choose something repetitive and heavy on documents that ends in a deliverable. Put the files in a folder, then write the outcome in one sentence along with the test you'll judge it by.

Stage the work

Connect only the folder the task needs. You can seed from an Anthropic industry pack so you start with proven structure.

Write the acceptance test

Decide what "done" means before you start. You'll reuse this same deliverable and test in Module 3 when you turn the job into a repeatable skill.

02 — Hand it off and steer

Sonnet, ask before acting, read the plan

Put Lesson 2 into practice. Start on Sonnet, keep approvals on, and read the plan before you let it run. Track what the task took so the value is easy to point to later.

Do

Hand off the whole task

Describe the outcome, let Cowork plan across the folder, approve the steps, and redirect with a sentence if it drifts. Don't reach in and take it over.

Direct the work; let Cowork do it.
Record

Time, rounds, and quality

How long it took, how many rounds of feedback, and how close the first pass came. These notes are what you'll show to prove value, and they feed the adoption metrics in Module 4.

A finished deliverable says more than a chat transcript.
03 — Debrief

Which part would you do every week?

The tasks you'd happily repeat are the ones to package as a skill next module. Name them now while the run is fresh.

01
Where it went smoothly
Which parts did Cowork handle with little steering? Those are your skill candidates.
02
Where it needed you
Where did you re-brief, add context, or correct it? Write it down, because good instructions in a skill are what fix it.
03
The job you'd repeat
Pick the one recurring task you'd most want to run the same way every time. That's what you turn into a skill in Module 3.
📝
Hold onto this

Keep your deliverable, your acceptance test, and your notes on what to improve. Module 3 turns this exact workflow into a reusable SKILL.md, so the value carries forward instead of resetting each time.

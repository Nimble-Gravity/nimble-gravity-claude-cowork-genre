Analytics & Adoption

Governance says the rollout is safe. Analytics says whether it's working — and whether it's paying for itself. Three surfaces give you the picture, each with a different job, and only one of them is close to session-level visibility.

Govern & Roll Out · ~20 min
01 — Three surfaces, three jobs

Know which one answers your question

Reaching for the wrong surface is how teams conclude "we can't measure this." Each covers a different altitude.

01
Admin dashboard — is anyone using it?
Cowork sessions, active users, and actions across a date range, refreshed at T+1, with CSV export. The fastest read on whether a pilot has a pulse. Start here in a steering meeting.
02
Analytics API — who, how deeply, at what cost?
Enterprise only. Per-user daily Cowork activity, org-wide DAU/WAU/MAU, skill and connector rankings by invocation, and per-user cost by model. This is where you find your champions and your Opus overuse.
03
OpenTelemetry — what actually happened?
Team and Enterprise. A real-time event stream into your SIEM — the closest thing to session-level visibility while Cowork sits outside the audit surfaces. Operational telemetry, not compliance-grade audit.
02 — The OpenTelemetry stream

Nothing flows until you turn it on

Configure it at Claude Desktop → Organization settings → Cowork: an OTLP endpoint URL, HTTP/JSON or HTTP/protobuf, and auth headers (encrypted at rest). Anthropic is explicit — "events are only exported when an admin configures an OTLP endpoint. No data flows by default."

What it emits

Six event families

User prompts (full text) · tool and MCP invocations, with server, tool, parameters, success or failure, and duration · file paths read or modified · skills and plugins invoked · human approval decisions — approved, rejected, or auto-initiated · API requests with model, token counts, cost estimates, and errors. Events share a prompt.id so a full response can be reconstructed.

Sinks: Splunk, Cribl, Elasticsearch, Loki, ClickHouse, Honeycomb, Datadog.
Handle with care

The stream carries prompt text

OTel exports prompt text and file paths to your collector. For a regulated reinsurer that is a data-handling decision in its own right — scope the sink, restrict who can query it, and write it into the control narrative. The upside: approval decisions in the stream are how you evidence that a human checkpoint existed.

A shared user account identifier correlates OTel events with Compliance API records.
03 — What to actually ask

Three questions, weekly then monthly

Anthropic's published rollout guidance reduces adoption measurement to three questions. Weekly active usage trending up is the headline metric.

Are people using it?

Weekly active users, and the trend. A department flat at zero isn't a tooling problem — it's an enablement problem, and it needs a champion, not a dashboard.

How deeply?

Dispatch turns per session and skill invocations separate real delegation from chat-with-extra-steps. Shallow usage means people are still treating Cowork like Chat.

Is it paying off?

Per-user cost by model against work that wasn't feasible before. This is where model-spend concentration shows up — and the fix is a cap on the Models tab, not a note in the minutes.

04 — Spend is a curriculum signal

The cost report is really a teaching list

This is the move most teams miss. When the same expensive operation keeps showing up, that isn't a billing problem to escalate — it's telling you exactly what to build or what to teach. Two routes out, and you pick per pattern.

Route 1 · Build a skill

Repetition is a skill candidate

Read the top expensive prompts card as a backlog, not a scoreboard. Run an analysis across it: is the same request being made often enough, by enough people, that it should stop being a prompt and become a skill? That is how the skills library grows on evidence instead of on volunteers.

Asked repeatedly → package it. That's the Workshop 3 loop, triggered by data.
Route 2 · Fix the instructions

Sometimes it's one line of CLAUDE.md

A session that makes hundreds of individual file edits — one tool call each — costs far more than the same work batched. You don't need a skill for that. You need an instruction in the user's CLAUDE.md asking for batched tool and file edits. Same output, a fraction of the spend.

Cheapest fix in the program: a sentence in a file.
🔁
Proactive vs. retroactive enablement

Retroactive is what most orgs do: someone runs up a bill, you have a conversation. Proactive is reading the pattern before it becomes a conversation — spotting that a whole desk is doing something the expensive way and shipping the skill or the instruction that fixes it for everyone at once. The analytics are what make proactive possible; without them you're only ever reacting to invoices.

05 — Build the view, then phase the rollout

From raw surfaces to a sponsor's dashboard

The native views get you started; the view leadership wants is one somebody has to assemble. We've already built it. Then RBAC lets you widen the rollout one group at a time.

You don't have to build this

The Nimble Gravity adoption dashboard

Assembling this yourself is a project — pulling the Analytics API and the OpenTelemetry stream into a BI layer, then designing the views. We ship it out of the box so a client can stand up Cowork analytics in days rather than a quarter. On the board: token usage, number of users, top expensive prompts, cost by model, API requests by model, reliability, the MCP servers in play, and daily prompts per user. Cowork and Claude Code appear as their own sections on the combined board — read the right section rather than looking for a tool filter.

Top expensive prompts is the card to spend time on — it's a backlog.
Phase it with RBAC

Pilot → department → org

Enable the Cowork capability for the pilot group only, then widen group by group as the numbers hold. Two to three champions per department, All-Staff 101s where everyone completes a real delegation that produces a deliverable, exec sponsors, and standing office hours.

The capability toggle is the rollout mechanism — not an email announcement.

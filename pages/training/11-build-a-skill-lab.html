Build a Skill Lab

The Skill Vault. A departing consultant locked the team's playbook behind four skill-locked doors, and their custom Cowork skills are the only keys. Work as a team, side by side with Cowork — every unlock code comes out of a real deliverable you produce — and escape before the 40-minute clock runs out.

Exercise · ~40 min
01 — The scenario

Escape the Skill Vault

The vault is a game; the work inside it is not. Each room hands you lab steps that you run in Cowork — invoking, repairing, authoring, and chaining skills — and the output of that work contains the code that opens the next door. If you treat Cowork like chat, you stay locked in.

Form a team of 2–4

One driver in Cowork, everyone on the puzzle. Enter a team name in the vault and the clock starts — total time, penalties included, goes on the leaderboard.

Vault + Cowork side by side

Keep the vault in one window and Cowork in the other. Each room's panel carries the lab steps and the code format; the facilitator hands out the lab file pack before the doors open.

Hints cost time

Two hints per room: the first is free, the second adds 2 minutes to your final time. Wrong codes cost nothing but pride — attempts are counted.

02 — The four rooms

Four doors, four skill disciplines

Each room exercises one thing Lessons 2 and 3 taught, in order of difficulty. By the last door you've touched the whole lifecycle: use, fix, build, combine.

01
The Inherited Skill
A working skill you've never seen: Meeting Extractor. Read its description, trigger it on real standup notes, and the action-item table it produces holds the first code. Proves: skills trigger on intent, not magic words.
02
The Broken Trigger
The Client Formatter skill never fires — its description was sabotaged down to "A skill." Rewrite it to say when to use it and what should trigger it, and the skill does the rest. Proves: the description is what makes a skill fire.
03
Forge Your Own Key
No skill exists for this door. Author a Candidate Evaluator from scratch — four required sections, a verdict constrained to three words — and run it on real interview notes. Proves: prescriptive instructions keep output from drifting.
04
The Playbook Door
Chain your Room 2 and Room 3 skills in one multi-intent prompt, then QA the result against a checklist. The deliverable — a client-ready hiring status update — carries the final code. Proves: skills compose, and QA closes the loop.
03 — What the vault is really teaching

The game is a wrapper for skill discipline

Every code is gated on the same standards Anthropic holds skills to — and Room 4's status update is a document you could actually send to a client.

Rooms 1–2 · Triggering

The description is the doorway

Room 1 makes you trigger a skill by matching its described intent; Room 2 makes you repair a description too vague to ever match. Together they teach the single highest-leverage line in any SKILL.md.

Skills trigger on intent, not magic words.
Rooms 3–4 · Authoring

Prescriptive beats vague

Room 3 only opens if your instructions enforce an exact output shape; Room 4 only opens if two skills fire from one prompt and survive a QA checklist. Vague instructions produce drifting formats — and locked doors.

Name the sections. Constrain the verdict. Then let the evals judge.
04 — After you escape (homework)

Ship a skill for your workflow

The vault used our scenario; homework uses yours. Take the workflow you brought to the session and run the skill-creator loop on it — evals before docs — then bring the packaged .skill to office hours.

01
Decompose it
Run the workflow-decomposition skill (download below). It interviews you about the steps, the data sources, and good vs bad output, then offers to draft your workflow as a skill.
02
Draft and test
Run the skill-creator loop from Make It a Skill: draft the SKILL.md, test against a baseline, and tune the description until it triggers reliably.
03
Package and bring it
Wrap it into a .skill and bring it to office hours. On Team and Enterprise it can then be shared as Personal, Shared, or Organization.

From Skills to Plugins

One skill helps a person. A plugin bundles skills, connectors, sub-agents, slash commands, and hooks into a single install you can hand to a whole team. It's the next layer up, and it's how skills get distributed.

Plugins & Rollout · ~20 min
01 — What a plugin bundles

Everything a workflow needs, in one install

A skill is a single capability. A plugin is the package around it: any mix of skills, MCP connectors, sub-agents, slash commands, and hooks, installed together.

What's inside

The skills hold the know-how, connectors bring in the data, sub-agents and slash commands carry out actions, and hooks handle automation. One install instead of a dozen setup steps.

Where it runs

Plugins run in Cowork and Claude Code, though not in Chat (bundled skills still work there). Hooks and sub-agents run only in Cowork. The layout puts plugin.json in .claude-plugin/ and skills/, agents/, and hooks/ at the root.

02 — Marketplaces

Install and adapt instead of starting blank

Anthropic ships built-in marketplaces, and you can add your own from GitHub. For our clients, the industry packs are the head start.

Built-in packs

Knowledge Work, Finance, Legal, Life Sciences

A Knowledge Work marketplace is there by default, along with Financial Services, Legal, and Life Sciences. You can add any GitHub repo as a marketplace by owner/repo or full URL.

Plugins are just markdown files, so they're easy to read and fork.
Portable by design

Author once in the Claude Code structure

Build in the Claude Code plugin structure — it's the superset, and the same plugin runs in Claude Cowork and Claude Code with nothing to rebuild. Keep the SKILL.md files tool-agnostic so they stay portable.

One source of truth; a fix ships to everyone at once.
03 — Design plugins to be trusted

Bundle tightly and watch what it can touch

Every connector and hook widens what a plugin can do, which is why restraint pays off before you push it to a team.

01
Scope to one job
A focused plugin for one workflow beats a sprawling one. Fewer, well-chosen parts are easier to trust, govern, and keep current.
02
Watch connectors and hooks
Connectors reach data and hooks automate actions. Stick to least privilege, gate the writes, and review what each part can do before it ships.
03
Keep skills tool-agnostic
So the same plugin runs in Claude Cowork and Claude Code without a rewrite, and stays simple to maintain.
🧩
Build a library you can reuse

Nimble Gravity's repeatable asset is a library of portable, well-scoped plugins by industry. Start each one in the Claude Code structure, keep the SKILL.md files tool-agnostic, and version them so a fix reaches every team at once.

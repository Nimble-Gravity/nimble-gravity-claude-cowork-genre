What You'll Build

Start with the model. End with a working app.

The capstone is a Loan Application Status Dashboard — the same data set used throughout the SDLC playbook. You scaffold it in Module 3 and complete it in Module 4, using only Claude Code prompts. By the end you have a filterable data table with status badges, a form, and clean HTML/CSS/JS your team can run.

Prereqs

A Claude.ai account (free tier works for Modules 1–2). Claude Code requires installation via npm. A text editor and a terminal. No prior AI experience required.

What you leave with

A mental model for tokens, context, and cost. Prompt patterns that transfer to any task. A configured Claude Code setup. A deployable capstone app built entirely with AI prompts.

Capstone App

Loan Application Status Dashboard

The app you build across Modules 3 and 4 — using nothing but Claude Code prompts.

01
Data table
Six branches from LOAN-2024-Q3 with Branch ID, Name, Target Approvals, Actual Approvals, % to Target, Last Updated, and Status. Fully rendered from a JS data array.
02
Status filter
Filter buttons for Ahead / On Track / At Risk. Click state handled in pure JS. Status badges use icon + label — never color alone — to meet color-blind accessibility requirements.
03
Add-row form
A modal form to add a new branch record. Data validation, form reset on close. Demonstrates that Claude Code can add interactive UI features in a single, well-scoped prompt.
How to use this workshop

Three things that will make this actually stick.

01
Do the exercises yourself
Watching someone else prompt Claude teaches you almost nothing. Every exercise page has copy-ready prompts. Run them. Watch what Claude does. Notice where it goes wrong — that's the real lesson.
02
Break things deliberately
After each exercise, try a deliberately bad prompt and compare the output. The gap between a weak prompt and a strong one will be obvious within two tries, and you'll never forget the difference.
03
Context is your main lever
By Module 3 it will be clear: Claude's capability is not the constraint. The quality and structure of what you give it is. This is true in a chat window and it is true in Claude Code.

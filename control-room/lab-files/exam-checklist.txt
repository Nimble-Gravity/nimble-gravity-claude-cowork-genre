INTERNAL AUDIT — COWORK CONTROL REVIEW
Evidence pack acceptance checklist  ·  9 items

Internal Audit will accept the one-page control narrative only if every item below passes.
Score each item PASS or FAIL, state briefly why, then report the total pass count.

------------------------------------------------------------------------------
 1. AUDIT COVERAGE STATED
    The document states explicitly that Claude Cowork activity is NOT covered by
    Anthropic's Audit Logs, Compliance API, or Data Exports, and that conversation
    history is stored locally on the user's machine.

 2. COMPENSATING ARCHITECTURE NAMED
    It describes the three monitoring planes and what each one covers:
    the Compliance API (control plane), OpenTelemetry (agent / operational plane),
    and a network proxy or LLM gateway (network / tool plane).

 3. HIGH-ASSURANCE WORK ROUTED AWAY
    It names at least one specific Gen Re workload that requires zero data retention
    or centralized audit, and routes that workload to an audited surface — the
    Anthropic API or Claude Code Enterprise — rather than the Cowork interface.

 4. LEAST PRIVILEGE APPLIED TO FOLDERS
    It states that folder scope is the operative data control, and gives a named
    example of a correctly scoped project folder rather than a drive or cloud root.

 5. ROLES DELEGATED, NOT BLANKETED
    Administration is delegated with Custom roles referencing at least one of the
    six admin permission areas (Identity & Access, Billing, Analytics, Privacy,
    User Management, Libraries) — not by granting Owner.

 6. EXPLICIT CAPABILITY GRANTS
    It states that Custom roles do not inherit organization-enabled capabilities,
    and that every capability a role needs must be granted explicitly.

 7. TELEMETRY CONFIGURED AND SCOPED
    It names the SIEM or collector that OpenTelemetry exports to, and acknowledges
    that no events flow until an admin configures an OTLP endpoint. It also notes
    that OTel carries prompt text and file paths, and says how that sink is scoped.

 8. ADOPTION MEASURED
    It names weekly active usage as the headline adoption metric and includes the
    three review questions: are people using it, how deeply, and is it paying off.

 9. OWNED AND RE-VERIFIED
    A named owner is accountable for the control narrative, and a re-verification
    cadence is stated for the audit-coverage gap.

------------------------------------------------------------------------------
VERDICT
    9 of 9 pass ............ READY
    7 or 8 pass ............ REMEDIATE
    6 or fewer ............. DEFER
